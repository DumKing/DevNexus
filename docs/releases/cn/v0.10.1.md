# DevNexus v0.10.1

[中文](./v0.10.1.md) | [English](../en/v0.10.1.md)

## 亮点

- 设置页重构为多 Tab：`基础设置` 和 `插件管理` 分离，语言、开机自启、更新检查、桌面通知、开发者日志不再与插件开关混在一起。
- 插件管理优化：隐藏插件内部版本展示，保留 manifest 版本字段用于后续单插件升级；新增搜索、分类筛选和启用数量统计，避免插件数量增加后列表过长。
- 新增开机自启开关：接入 Tauri autostart 插件，可在设置中控制 DevNexus 是否随系统登录启动。
- LAN Chat 通知增强：接入 Tauri 原生 notification 插件，桌面端优先使用系统通知；收到未读消息时会请求窗口注意力，Windows 任务栏图标可闪烁提醒。
- Windows 托盘行为：最小化仍进入任务栏；关闭按钮隐藏到系统托盘；托盘菜单提供“打开 DevNexus”和“退出”，真正退出通过托盘菜单完成。
- S3/OSS 连接列表改为可折叠分组，组内连接横向卡片排列；编辑 S3 连接时保留原连接 id，避免保存后变成新连接。
- Redis、SSH、MongoDB、MySQL、S3 连接列表体验统一为可折叠分组 + 横向卡片；Redis/SSH 编辑连接时留空凭据会保留原密码/密钥口令。
- 新增插件级 i18n 基础能力：主应用提供当前语言和翻译 hook，插件可在各自目录维护语言包，便于后续插件市场和单插件交付。

## 验证

- `npm test`
- `npm run build`
- `cd src-tauri && cargo check`

## 已知限制

- 插件版本字段目前仅作为后续单插件升级预留，不在设置页展示，也不支持单插件在线升级。
- LAN Chat 系统通知依赖操作系统通知权限；若系统级通知被禁用，DevNexus 只能更新未读角标和窗口注意力状态。
- 托盘常驻行为当前优先适配 Windows 桌面端；macOS/Linux 的托盘展示受系统桌面环境限制。
