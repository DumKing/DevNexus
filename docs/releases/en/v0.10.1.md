# DevNexus v0.10.1

[中文](../cn/v0.10.1.md) | [English](./v0.10.1.md)

## Highlights

- Settings now use multiple tabs: `Basic` and `Plugins` are separated, so language, startup launch, update checks, desktop notifications, and developer logs no longer compete with plugin toggles.
- Plugin management is more scalable: plugin versions are hidden from the settings UI while remaining in manifests for future per-plugin upgrades; search, category filters, and enabled counts make the list easier to manage as more plugins are added.
- Added startup launch control via the Tauri autostart plugin.
- LAN Chat notifications now use Tauri native notifications on desktop first, with the Web Notification API kept as a browser fallback; new unread messages also request window attention so the Windows taskbar can flash.
- Windows tray behavior: minimize still goes to the taskbar, close hides DevNexus to the system tray, and the tray menu provides `Open DevNexus` and `Exit`.
- S3/OSS connection lists now use collapsible groups with horizontal connection cards; editing an S3 connection preserves the original connection id instead of creating a new profile.
- Redis, SSH, MongoDB, MySQL, and S3 connection pages now share the collapsible grouped-card layout; Redis and SSH keep the saved password/passphrase when editing metadata with credential fields left blank.
- Added plugin-level i18n foundations: the app shell provides current language and translation hooks while plugins can keep their own language packs, preparing for future plugin-market delivery.

## Validation

- `npm test`
- `npm run build`
- `cd src-tauri && cargo check`

## Known Limits

- Plugin manifest versions are currently reserved for future per-plugin upgrades; they are not shown in Settings and individual plugin online upgrades are not implemented yet.
- LAN Chat system notifications depend on OS notification permission. If disabled at the OS level, DevNexus still updates unread badges and window attention state.
- Tray behavior is primarily targeted at Windows desktop for this release; macOS/Linux tray visibility depends on the desktop environment.
